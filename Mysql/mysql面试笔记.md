## MVCC-多版本并发控制
MVCC 是机制，undo log实现，可以理解为MVCC是接口，undo log是实现

- **MVCC（多版本并发控制）**
    
    让读写不阻塞、读读不阻塞，实现**非锁定一致性读**（比如普通 `SELECT`）。
    
    核心思路：**一条记录保留多个历史版本，事务读自己可见的那个版本**。
    
- **undo log（回滚日志）**
    
    记录数据修改前的旧值，作用有两个：
    
    1. 事务回滚时，用旧值把数据恢复原样；
    2. 给 MVCC 提供**历史版本链**。

undo日志：实现事务的原子性，回滚，按事务id顺序通过roll_pointer生成日志链表
redo_log：崩溃恢复，事务的持久性，ib_logfile0和 ib_logfile1的文件，这个就是redo日志，空间小，顺序写入，记录事务过程中所有操作

可重复读：使用读视图readView，每次读取这个视图

幻读
就是第一次读如果 是空的情况，且在自己事务中进行了该条数据的修改

binlog 是用作人工恢复数据，redo log 是 MySQL 自己使用，用于保证在数据库崩溃时的 事务持久性
